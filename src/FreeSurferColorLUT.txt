<h1 id="aparcstats2table-1">aparcstats2table</h1>
<span class="anchor" id="line-2"></span><p class="line867">
<h2 id="Introduction">Introduction</h2>
<span class="anchor" id="line-3"></span><p class="line874">This script will generate text/ascii tables of freesurfer parcellation stats data, ?h.aparc.stats. This can then be easily imported into a spreadsheet and/or stats program. <span class="anchor" id="line-4"></span><span class="anchor" id="line-5"></span><p class="line867">
<h2 id="Usage">Usage</h2>
<span class="anchor" id="line-6"></span><p class="line874">Converts a cortical stats file created by recon-all and or mris_anatomical_stats (eg, ?h.aparc.stats) into a table in which each line is a subject and each column is a parcellation. By default, the values are the area of the parcellation in mm2. The first row is a list of the parcellation names. The first column is the subject name. If the measure is thickness then the last column is the mean cortical thickness. <span class="anchor" id="line-7"></span><span class="anchor" id="line-8"></span><p class="line874">The subjects list can be specified on either of two ways: <span class="anchor" id="line-9"></span><span class="anchor" id="line-10"></span><ol type="1"><li>Specify each subject after a -s flag <span class="anchor" id="line-11"></span><ul><li style="list-style-type:none">-s subject1 -s subject2 ... --hemi lh <span class="anchor" id="line-12"></span><span class="anchor" id="line-13"></span></li></ul></li><li class="gap">Specify all subjects after --subjects flag. --subjects does not have <span class="anchor" id="line-14"></span><ul><li style="list-style-type:none">to be the last argument. Eg: <span class="anchor" id="line-15"></span><ul><li style="list-style-type:none">--subject subject1 subject2 ... --hemi lh <span class="anchor" id="line-16"></span><span class="anchor" id="line-17"></span></li></ul></li></ul></li></ol><p class="line874">By default, it looks for the ?h.aparc.stats file based on the Killiany/Desikan parcellation atlas. This can be changed with '--parc parcellation' where parcellation is the parcellation to use. An alternative is aparc.a2005s which was developed by Christophe Destrieux. If this file is not found, it will exit with an error unless --skip in which case it skips this subject and moves on to the next. <span class="anchor" id="line-18"></span><span class="anchor" id="line-19"></span><p class="line874">By default, the area (mm2) of each parcellation is reported. This can be changed with '--meas measure', where measure can be area, volume (ie, volume of gray matter), thickness, thicknessstd, or meancurv. thicknessstd is the standard dev of thickness across space. <span class="anchor" id="line-20"></span><span class="anchor" id="line-21"></span><div><table><tbody><tr>  <td><p class="line862"> --version </td>
  <td><p class="line862">  show program's version number and exit </td>
</tr>
<tr>  <td><span class="anchor" id="line-22"></span><p class="line862"> -h, --help </td>
  <td><p class="line862"> show this help message and exit </td>
</tr>
<tr>  <td><span class="anchor" id="line-23"></span><p class="line862"> --subjects (REQUIRED) subject1 &lt;subject2 subject3..&gt;, -s SUBJECTS subjectname, --subjectsfile=SUBJECTSFILE </td>
  <td><p class="line862"> name of the file which has the list of subjects (one subject per line) </td>
</tr>
<tr>  <td><span class="anchor" id="line-24"></span><p class="line862"> --qdec=QDEC </td>
  <td><p class="line862"> name of the qdec table which has the column of subjects ids (fsid) </td>
</tr>
<tr>  <td><span class="anchor" id="line-25"></span><p class="line862">  --qdec-long=QDECLONG </td>
  <td><p class="line862"> name of the longitudinal qdec table which has the column of tp ids (fsid) and subject templates (fsid- base) </td>
</tr>
<tr>  <td><span class="anchor" id="line-26"></span><p class="line862">  --hemi=HEMI (REQUIRED) </td>
  <td><p class="line862"> lh or rh </td>
</tr>
<tr>  <td><span class="anchor" id="line-27"></span><p class="line862"> -t OUTPUTFILE, --tablefile=OUTPUTFILE (REQUIRED) </td>
  <td><p class="line862"> output table file </td>
</tr>
<tr>  <td><span class="anchor" id="line-28"></span><p class="line862"> -p PARC, --parc=PARC </td>
  <td><p class="line862"> parcellation. default is aparc (alt: aparc.a2009s)</td>
</tr>
<tr>  <td><span class="anchor" id="line-29"></span><p class="line862"> -m MEAS, --measure=MEAS </td>
  <td><p class="line862"> measure: default is area ( alt volume, thickness, thicknessstd, meancurv, gauscurv, foldind, curvind)</td>
</tr>
<tr>  <td><span class="anchor" id="line-30"></span><p class="line862">  -d DELIMITER, --delimiter=DELIMITER </td>
  <td><p class="line862"> delimiter between measures in the table. default is tab (alt: comma, space, semicolon ) </td>
</tr>
<tr>  <td><span class="anchor" id="line-31"></span><p class="line862">  --skip  </td>
  <td><p class="line862"> if a subject does not have input, skip it.  Skips if it can't find a .stats file.  Default behavior is to exit the program. </td>
</tr>
<tr>  <td><span class="anchor" id="line-32"></span><p class="line862"> --parcid-only </td>
  <td><p class="line862"> do not pre/append hemi/meas to parcellation name </td>
</tr>
<tr>  <td><span class="anchor" id="line-33"></span><p class="line862"> --common-parcs </td>
  <td><p class="line862"> output only the common parcellations of all the subjects given </td>
</tr>
<tr>  <td><span class="anchor" id="line-34"></span><p class="line862"> --parcs-from-file=PARCSFILE </td>
  <td><p class="line862"> filename: output parcellations specified in the file </td>
</tr>
<tr>  <td><span class="anchor" id="line-35"></span><p class="line862">  --report-rois </td>
  <td><p class="line862">  print ROIs information for each subject, gives what ROIs that are present in at least one other subject is absent in current subject and also gives what ROIs are unique to the current subject  </td>
</tr>
<tr>  <td><span class="anchor" id="line-36"></span><p class="line862"> --transpose </td>
  <td><p class="line862"> transpose the table ( default is subjects in rows and ROIs in cols).  This might be a useful way to see the table when the number of subjects is relatively less than the number of ROIs. </td>
</tr>
<tr>  <td><span class="anchor" id="line-37"></span><p class="line862"> -v, --debug  </td>
  <td><p class="line862"> increase verbosity </td>
</tr>
</tbody></table></div><span class="anchor" id="line-38"></span><span class="anchor" id="line-39"></span><p class="line867">
<h2 id="SampleCommandline">Sample Command line</h2>
<span class="anchor" id="line-40"></span><p class="line867"><span class="anchor" id="line-41"></span><span class="anchor" id="line-42"></span><pre>aparcstats2table --subjects bert ernie fred margaret --hemi rh --meas thickness --tablefile aparc_stats.txt</pre><span class="anchor" id="line-43"></span><p class="line862">This will combine the <tt>rh.aparc.stats</tt> files for the subjects <tt>bert,&nbsp;ernie,&nbsp;fred,</tt> and <tt>margaret</tt> to generate one table, <tt>aparc_stats.txt</tt>, that will report the thickness of all the structures labeled in rh.aparc.annot.  The resulting file can be imported into any spreadsheet program. <span class="anchor" id="line-44"></span><span class="anchor" id="line-45"></span><p class="line862">If you'd rather get your information from the alternate parcellation, ?h.aparc.a2005s.annot, you can specify this on your command line with <tt>&nbsp;--parc&nbsp;aparc.a2005s</tt>.  Also, if you'd like something other than thickness you can specify this with <tt>--meas</tt> and specify area, volume, or meancurv. <span class="anchor" id="line-46"></span><span class="anchor" id="line-47"></span><p class="line867"><span class="anchor" id="line-48"></span><span class="anchor" id="line-49"></span><pre> aparcstats2table --hemi lh --subjects 004 008 --parc aparc.a2005s --meas meancurv --tablefile lh.a2005s.meancurv.txt</pre><span class="anchor" id="line-50"></span><p class="line874">lh.a2005s.meancurv.txt will have 3 rows: (1) 'header' with the name of each structure, (2) mean curvature for each structure for subject <span class="anchor" id="line-51"></span><span class="anchor" id="bottom"></span></div>
